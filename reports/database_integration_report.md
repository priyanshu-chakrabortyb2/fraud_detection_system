# Phase 7: Database Integration

## Objective

The objective of this phase was to integrate persistent storage into the Fraud Detection System. Instead of returning predictions temporarily, every prediction generated by the API is now stored in a database for future retrieval, auditing, and analysis.

This phase enables prediction history tracking and forms the foundation for monitoring and dashboard development in later phases.

---

* SQLite
* SQLAlchemy
* FastAPI
* Python

# Database Configuration

The project uses SQLite as the database engine.

Database File:

```text
fraud_detection.db
```

Advantages:

* Lightweight
* No separate database server required
* Easy deployment
* Suitable for MVP development
* Fully compatible with SQLAlchemy

---

# Database Model

A Prediction table was created to store all fraud prediction results.

Schema:

| Column            | Data Type |
| ----------------- | --------- |
| id                | Integer   |
| prediction        | String    |
| fraud_probability | Float     |
| risk_score        | Integer   |
| risk_category     | String    |
| reasons           | Text      |

---

# CRUD Operations

The following database operations were implemented.

## save_prediction()

Purpose:

Stores prediction results in the database.

Stored Information:

* Prediction Label
* Fraud Probability
* Risk Score
* Risk Category
* SHAP-Based Explanations

---

## get_predictions()

Purpose:

Retrieves all stored predictions.

Used By:

```text
GET /history
```

---

## get_prediction_by_id()

Purpose:

Retrieves a specific prediction record.

Used By:

```text
GET /history/{id}
```

# FastAPI Integration

Workflow:

```text
Transaction Input
        |
        v
Random Forest Prediction
        |
        v
Risk Calculation
        |
        v
SHAP Explanation
        |
        v
Save Prediction To Database
        |
        v

```

Every API prediction request is automatically logged.

# History Endpoints

## GET /history

Purpose:

Retrieve all stored prediction records.

Example Response:

```json
[
  {
    "id": 1,
    "prediction": "Fraud",
    "fraud_probability": 0.9998,
    "risk_score": 100,
    "risk_category": "High Risk"
  }
]
```

---

## GET /history/{id}

Purpose:

Retrieve a specific prediction by ID.

Example Response:

```json
{
  "id": 1,
  "prediction": "Fraud",
  "fraud_probability": 0.9998,
  "risk_score": 100,
  "risk_category": "High Risk",
  "reasons": "..."
}
```
